# S0 编排者指令：发现与确认（Discovery）

## 角色
在开始任何处理之前，**先确认**。不确定的信息不进入 S1。

## 执行流程

### 步骤 1：确认输入
- [ ] SRS 文件路径存在且可读？
- [ ] 文件格式识别（.md / .html / 多目录包）？
- [ ] 文件大小和行数估算？

### 步骤 2：内容扫描（不修改任何文件）
- [ ] 检测 §7 未解决问题：有___条，优先级 P0
- [ ] 检测术语表（§1.4 或 Glossary）：存在 / 缺失
- [ ] 检测模块能力矩阵（§2.9）：存在 / 缺失
- [ ] 检测测试用例章节（§5）：有___条测试用例
- [ ] 粗略章节数：约___个一级标题

### 步骤 3：触发条件检测

**TLA+ 触发检测**（扫描以下关键词）：
- [ ] 微服务协作 / 并行进程 / 分布式锁 / 资源争抢
- [ ] 分布式事务 / 共识协议 / 跨服务状态机
- [ ] 检测结果：触发 / 不触发

**Lean 4 触发检测**（扫描以下关键词）：
- [ ] 非常见算法 / 安全关键 / 密码学协议
- [ ] 金融核心 / 复杂调度 / 自定义数据结构
- [ ] 检测结果：触发 / 不触发

### 步骤 4：向用户报告并确认

输出：
```
## SRS 分析报告

- 文件：<path>（<N>行，<格式>）
- 章节：约<M>个一级标题
- 缺口：§7 有<K>条未解决问题，术语表<存在/缺失>

### 建议的阶段触发
- S1-S4（必选）：预处理 → 需求提取 → 图谱构建 → BDD 生成
- S5 TLA+：<触发/跳过>（原因：<检测到的关键词>）
- S5 Lean 4：<触发/跳过>（原因：<检测到的关键词>）

### 预估
- 分片数：约<N>个
- 建议语言：<zh/en>

是否继续？可回复：
- "继续" → 进入 S1
- "跳过 S5" → 跳过形式化阶段
- "仅 S1-S3" → 只做到图谱导出
```

### 步骤 5：等待用户确认后进入 S1
**未确认前禁止执行 init 或任何文件写入操作。**
