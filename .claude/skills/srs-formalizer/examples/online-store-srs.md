## §2 系统概述 / System Overview

### §2.1 系统上下文
在线商城系统是一个面向消费者的B2C电子商务平台，支持用户注册、商品浏览、下单购买、在线支付等完整购物流程。

### §2.9 模块能力矩阵 / Module Capability Matrix

| 模块 / Module | 能力 / Capabilities |
|--------------|---------------------|
| 用户模块 / User | 注册、登录、个人信息管理、地址管理 |
| 商品模块 / Product | 商品浏览、搜索、分类、库存展示 |
| 订单模块 / Order | 创建订单、订单查询、订单取消、退款申请 |
| 支付模块 / Payment | 微信支付、支付宝支付、支付状态回调 |
| 库存模块 / Inventory | 库存锁定、库存扣减、库存释放 |

## §3 功能需求 / Functional Requirements

### §3.1 用户模块 / User Module

#### §3.1.1 用户注册 / User Registration
系统应支持手机号注册和邮箱注册两种方式。
- 手机号注册需发送短信验证码，验证码有效期5分钟
- 邮箱注册需发送验证邮件，链接有效期24小时
- 密码长度不少于8位，必须包含字母和数字

#### §3.1.2 用户登录 / User Login
系统应支持密码登录和短信验证码登录。
- 连续5次登录失败后，账户锁定15分钟
- 登录成功后生成JWT Token，有效期7天

#### §3.1.3 个人信息管理 / Profile Management
用户可以查看和修改个人资料，包括昵称、头像、收货地址。

### §3.2 商品模块 / Product Module

#### §3.2.1 商品浏览 / Product Browsing
用户可以按分类浏览商品，查看商品详情、价格、库存、评价。
- 商品列表支持分页，每页20条
- 支持按价格、销量、好评率排序

#### §3.2.2 商品搜索 / Product Search
用户可以通过关键词搜索商品，支持按分类、价格区间筛选。

### §3.3 订单模块 / Order Module

#### §3.3.1 创建订单 / Create Order
用户选择商品后可创建订单，创建订单时系统应锁定对应库存。
- 库存锁定超时时间为30分钟
- 超时未支付自动取消订单并释放库存

#### §3.3.2 支付订单 / Pay for Order
系统应支持微信支付和支付宝支付两种支付方式。
- 支付成功后更新订单状态为"已支付"
- 扣减实际库存
- 触发订单履约流程

#### §3.3.3 订单取消 / Cancel Order
未支付订单用户可主动取消，已支付订单需申请退款。

### §3.4 支付模块 / Payment Module

#### §3.4.1 支付回调 / Payment Callback
系统接收第三方支付平台的异步回调通知，验证签名后更新支付状态。
- 回调需验证签名防止伪造
- 支持支付成功、支付失败、退款回调

### §3.5 库存模块 / Inventory Module

#### §3.5.1 库存锁定 / Inventory Lock
创建订单时预锁定库存，使用Redis分布式锁防止超卖。

#### §3.5.2 库存释放 / Inventory Release
订单取消或支付超时后，释放已锁定的库存。

## §4 非功能需求 / Non-Functional Requirements

### §4.1 性能需求 / Performance
- 商品列表接口响应时间 < 200ms（P95）
- 订单创建接口响应时间 < 500ms（P95）
- 系统支持峰值TPS ≥ 1000
- 支持同时在线用户数 ≥ 10万

### §4.2 安全需求 / Security
- 用户密码必须使用bcrypt加密存储
- 所有支付接口必须签名验证
- 防止SQL注入、XSS、CSRF攻击
- 敏感操作需二次验证

### §4.3 可用性需求 / Availability
- 系统年可用性 ≥ 99.9%
- 计划内停机时间每月不超过4小时
- 故障恢复时间RTO < 30分钟
- 数据备份RPO < 5分钟

### §4.4 可维护性需求 / Maintainability
- 代码覆盖率 ≥ 80%
- API文档自动生成
- 关键操作日志完整记录

## §5 业务规则 / Business Rules

1. 同一用户同一商品24小时内最多下单3次（防刷单）
2. 订单支付超时时间为30分钟
3. 退款申请需在收货后7天内提出
4. 虚拟商品不支持7天无理由退款

---

## §6 约束 / Constraints

1. 后端技术栈：Node.js + TypeScript
2. 数据库：PostgreSQL 14+
3. 缓存：Redis 6+
4. 消息队列：RabbitMQ
5. 部署环境：Linux (x86_64)

---
